## תהליך עיבוד הנתונים (Preprocessing)

העיבוד מתבצע ב-`src/data_processing.py` לכל מטופל בנפרד, לפי הסדר הבא:

### שלב 1 — טעינת נתונים
- קריאת אות ECG מהקובץ `{patient}.csv`
- קריאת annotations מ-`{patient}annotations.txt`

### שלב 2 — סינון רעש (Filtering)
- **Bandpass Butterworth filter**
- תדרים: **0.4–30 Hz**
- קצב דגימה: **360 Hz**
- סder: 3
- מטרה: הסרת baseline wander ורעשים בתדרים גבוהים

### שלב 3 — Normalization
- נרמול **Min-Max** לטווח **[0, 1]** לכל מטופל בנפרד
- נוסחה: `(signal - min) / (max - min)`

### שלב 4 — Segmentation (חלוקה לחלונות זמן)
- חלוקת האות לחלונות של **שנייה אחת** (360 דגימות)
- לכל חלון — תיוג בינארי לפי annotations:
  - **Normal (0):** beat מסוג `N`
  - **Abnormal (1):** `L`, `R`, `V`, `A`, `a`, `j`, `S`, `F`, `E`, `e`, `r`, `!`, `f`, `/`
  - beats לא מסווגים (`Q`, `?`) — **נזרקים**
- אם בחלון יש לפחות beat abnormal אחד → החלון מתויג כ-Abnormal

### שלב 5 — Resampling
- כל חלון מושם לגודל קבוע: **128 דגימות**
- נדרש כדי שכל הקלט  יהיה באורך אחיד ל-CNN

### שלב 6 — Reshape ל-CNN
- המרה מ-`(N, 128)` ל-`(N, 1, 128)` — ערוץ אחד, אורך 128

### שלב 7 — סינון מטופלים
מטופלים עם חוסר איזון קיצוני (< 0.9% abnormal) **מוצאים מהדאטה**:

```
101, 103, 107, 109, 111, 112, 113, 115, 117, 118,
121, 122, 123, 124, 207, 214, 230, 232, 234
```

**19 מטופלים הוצאו** → **29 מטופלים** נשארו לאימון.

## חלוקת הדאטה סט

### עקרון החלוקה
החלוקה מתבצעת **לפי מזהה מטופל (Patient ID)** — לא אקראית לפי beat.

> **חשוב:** כל ה-beats של מטופל נמצאים **רק** באחד מה-splits.  
> מטופלי Test **לא** הופיעו באימון או ב-Validation — מונע data leakage.

| **Test** | ~17.2% (5/29) | 5 |

**Seed:** 42 (לשחזוריות)  
**פרמטרים:** `--val_ratio 0.15`, `--test_ratio 0.15`

## ארכיטקטורת המודל

**סוג:** 1D Convolutional Neural Network (ResNet-style blocks)  
**קובץ:** `src/models/cnn1D.py`

| רכיב | פירוט |
|------|--------|
| **Input** | `(batch, 1, 128)` — חלון ECG בודד |
| **Conv1d ראשוני** | 1 → 32 channels, kernel=1 |
| **Blocks** | 4 blocks (Conv1d × 2 + skip connection + MaxPool1d) |
| **Block channels** | 32 |
| **Kernel size** | 5 |
| **Flatten** | → 160 features |
| **Linear 1** | 160 → 160 |
| **Linear 2** | 160 → 1 |
| **Output** | Sigmoid → הסתברות [0, 1] |
| **Threshold** | ≥ 0.5 → Abnormal, < 0.5 → Normal |

### Loss Function
- **Binary Cross Entropy Loss (BCELoss)**

### פרשנות

**נקודות חוזק:**
- **Recall גבוה (93.3%)** — המודל תופס את רוב הדופקים החריגים.  
  רק **155 מתוך 2,299** abnormal beats פוספסו — חשוב ביישום רפואי.
- **Accuracy 84%** — ביצועים סבירים על מטופלים חדשים לגמרi.

**נקודות לשיפור:**
- **Precision בינוני (64%)** — כ-36% מהחיזויים "Abnormal" הם false positives.  
  1,211 דופקים תקינים סווגו בטעות כחריגים.
- **Overfitting** — Train accuracy הגיע ל-~99.9% בזמן ש-Test accuracy 84%.  
  המודל "זוכר" את מטופלי האימון יותר מדי.

## מסקנות

1. **כל 5 דרישות הפרויקט ממומשות** — סיווג בינארי, preprocessing מלא, MIT-BIH, חלוקה ל-3 splits, 1D CNN.

2. **המודל מתכלל למטופלים חדשים** — 84% accuracy על 5 מטופלים שלא נראו באימון.

3. **Recall גבוה (93%)** — המודל מתאים לתרחיש שבו חשוב לא לפספס דופקים חריגים.

4. **Precision נמוך יחסית (64%)** — יש false positives; ניתן לשפר עם threshold tuning, class weights, או regularization.

5. **Overfitting** — המודל נבחר ב-epoch 5 (early best val loss); epochs מאוחרים יותר לא שיפרו generalization.

6. **המלצות לשיפור:**
   - Early stopping אוטומati ב-epoch 5
   - Dropout / regularization
   - Threshold optimization (לא רק 0.5)
   - Data augmentation
   - Cross-validation על חלוקות מטופלים שונות
